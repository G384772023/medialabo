数当てゲーム

この数当てゲームは数字を予想して3回以内に正解するというものです。遊び方はまず、入力できる場所に１〜１０の好きな数字を入力します。 そうすると、その下に正解だった場合「正解！」とでて、間違いだった場合答えが予想よりも大きいか小さいかを教えてくれます。それを元に２、3回目をまた入力します。 もし3回とも間違えてしまった場合は答えが表示されて終了します。また、4回目以降入力しても終了しているため答えをずっと表示し続けます。 また、3回以内に正解した場合も終了してその後何回やっても同じ表示がされます。もう一度挑戦される場合は読み込みをしてください。その時は答えも前回とは違っているので何度でも楽しむことができます。

数当てゲームのページ:kazuate.html

各授業回の学習事項

第1回 WWW と HTML

HTML文書のp要素やstrong要素、a要素、ハイパーリンクの書き方、属性などのHTML文書の基本的な書き方とHTMLの書き方や新しい要素を知りたい時の便利なサイトがあること、エラーやバグが起きた時のデバッグのやり方を教わりました。

演習の成果:演習1-4:HTMLのさまざまな要素

第2回 CSS による Web ページの装飾

CSSの説明やセレクタ、プロパティ名、プロパティ値などの書式の説明、他にもWeb開発ツールを使ったデバックのやり方などを教わった。Box pなどとスペースが空いてるのは前に書かれている中の後ろの要素などという意味があるということなども教わった。

演習の成果:練習2-3:CSSによる編集

第3回 JavaScript の基本

JavaScriptの主な書き方やデバックの仕方などを学びました。他にもjavaとの違いとしてコンパイルの有無やそれによってのエラーの見分け方の難しさなども知りました。それと、JavaScriptのことを調べるときはグーグルなどで調べると昔のと混合していることがあることからMDN Web Docsから得るのが良い。

演習の成果:練習3-6:JavaScriptの構文

第4回 DOM と JavaScript

DOMは今までに習ったHTMLやCSSなどの内容をJavaScriptなどのプログラミングによって内容を足したり、消したり、変更したりできます。でも、変更するなら元のプログラミングを変えた方が良いと思うかもしれませんが、これの強みは書かれたものを後から変えることができることにあり、第5回の内容から便利なものとなる。

演習の成果練習4-2~5:DOMとjavaScript

第5回 イベント処理

イベント処理とはボタンをクリックしたり、マウスを動かしたりなどの動作に反応して動くプログラミングです。これと第4回の内容であるDOMを組み合わせると元々作ってあるHTMLの内容を後から変えることができます。こうすることで他から得た情報を時間ごとに変えることができたりもします。

演習の成果練習5-3:イベント処理

感想

自分はこの講義で多くのことを学びました。最初の方はHTMLや要素など全くわからずにただ講義の資料に書いてあるものを見様見真似で書いていました。それでもうまくいかない場合はSTの方に聞いたり、友人に教えてもらったりしました。その結果、自分なりにそれぞれを解釈して、理解することができました。特に難しいと思ったのはDOMの回です。 DOMは確かに便利で色々なことができますが、その色々できるということから覚えるものが多く、うまく動作しないことが多々ありました。でもなんとか使えるところまで理解することができました。また、難しかったことで言うと数当てゲームも大変でした。数字を打って、イベントのボタンを押してもなんの変化もなかったり、 答えの数字を打ったのになぜか正解が出なかったりとバグがたくさんでて苦戦しました。工夫したところは講義資料では『~回目の予想:』の~が正解した時にいきなり増えていたので自分なりに工夫をして、そこを修正したりしました。プロジェクト課題は天気を表示するプログラミングをやりました。ここで一番苦戦したのはデータの出し方でした。 どの操作がどういう動きをしているかがうまくわからなかったので、データを取り出すとなった時何をすればいいかがうまくわからずに苦戦しました。でもこの時に便利だと思ったのは講義で紹介されていたコンソールでのやり方とデバッガーのエラーの見分け方でした。コンソールのやり方は最初の方はいちいち書いてそれをコピーして貼るという二度手間で面倒臭いと思っていました。 でも詰まってしまってどうすればいいかわからない時は直接書くよりも簡単で見たい情報をすぐに見られるという利点がとても便利だと思いました。また、デバッガーもうまくいかない時にどこがエラーになっているかをそこで判断できてすぐに直すことができました。最後の第6回のAJAX通信では通信がうまくできているかの確認がわかりやすく、すぐに修正ができた。 今回のWebコンテンツ作成で１から作ってみて、難しくて心が折れそうになったこともありますが、企業などでも１から作るということはないとのことなのでこれができて理解できたということは今後かなりのアドバンテージになると自分は思っています。この知識があるのとないのではこのような仕事をするうえで理解度にかなり差がでて来ると思います。 自分の中での職業の選択肢も増やすことができて、自分にとっては良いことづくしなので、この講義はとてもよかったと思いました。
